# 產品需求文件 (PRD) - 個人記帳簿系統

## 1. 專案概述
- **背景與動機**：現代人生活忙碌，常常不清楚自己的金流去向。建立一個操作簡便的個人記帳簿系統，能幫助使用者輕鬆紀錄並追蹤日常收支，進而達成財務管理目標。
- **目標用戶**：學生、成年人等有記帳與理財需求的一般大眾。
- **核心價值主張**：提供一個可以「完美紀錄收入支出」的系統，透過直覺的介面與清晰的統計報表，讓使用者無痛養成記帳習慣，輕鬆掌握個人財務狀況。

## 2. 功能需求
以下列出系統的主要功能與對應的使用者故事（User Story）：

1. **紀錄收入與支出（含細項）**
   - 作為 **使用者**，我希望 **能快速新增一筆收入或支出，並填寫金額、日期與細項備註**，以便 **完整紀錄我的每一筆金流**。
2. **收支分類管理**
   - 作為 **使用者**，我希望 **能為收支設定不同的分類（如餐飲、交通、薪水等）**，以便 **日後能依據分類檢視花費比例**。
3. **顯示統計資訊（月次/圖表統計）**
   - 作為 **使用者**，我希望 **能查看每月或特定期間的收支統計與圖表**，以便 **清楚了解自己的花費趨勢與結餘（對應需求：顯示月次統計）**。
4. **預算設定與提醒**
   - 作為 **使用者**，我希望 **能針對特定分類或總金額設定每月預算**，以便 **在快超支時能控制自己的花費**。
5. **歷史紀錄查詢與篩選**
   - 作為 **使用者**，我希望 **能依據日期、分類或關鍵字搜尋過去的記帳紀錄**，以便 **快速找到特定的花費明細**。

## 3. 非功能需求
- **技術限制**：系統預計使用 Python Flask 作為後端框架，搭配 Jinja2 作為前端模板引擎，並使用 SQLite 作為關聯式資料庫。
- **效能考量**：頁面載入時間應小於 2 秒，確保使用者記帳時能獲得流暢的體驗；資料庫查詢需建立適當的索引以加速統計報表的生成。
- **安全考量**：需實作基本的使用者登入與註冊機制（密碼需加密儲存），確保每位使用者的記帳資料具備隱私性且不被他人存取；防範常見的 Web 攻擊（如 SQL Injection、XSS 等）。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 使用者註冊與登入
  - 新增、修改、刪除收入與支出紀錄（含細項）
  - 預設的基本收支分類
  - 檢視當月收支總計與明細列表
- **Should Have (應該有)**：
  - 基礎統計圖表（如圓餅圖顯示各分類花費比例）
  - 自訂收支分類
- **Nice to Have (可以有)**：
  - 預算設定與超支提醒功能
  - 匯出記帳資料（如 CSV 或 Excel 格式）
  - 支援多種幣別或自動記帳提醒
